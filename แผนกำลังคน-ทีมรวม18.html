6สิ่งที่ต้องเคาะ/เจรจา (เรียงความสำคัญ)

  1. ประกบจุดอ่อน 2 จุดของทีมผมให้ชัดก่อน:
    • ชมพู่ BA ไม่รู้ flow → ล็อก ก้อง/โอ/สาม มานำ Fit/Gap ช่วง ก.ค.–ต.ค. (ทั้งสามดึงยาก-ปานกลาง → ต้องจองเร็ว) + ชมพู่ shadow เก็บความรู้
    • จูน PO เพิ่งเข้า → แผน ramp-up 2–4 สัปดาห์ + มีน เป็นพี่เลี้ยง + อาร์ต ประกบช่วงตัดสินใจสำคัญ
  2. ล็อก 5 บทบาทที่ทีมผมไม่มีเลย — Migration จีเน่ (ดึงยาก ★เสี่ยงสุด), Integration พีช (ดึงยากมาก), QA จ๊อบ/ไอซ์ (ดึงง่าย), UX/UI มิ้น/ณัฐ, DB หมี · จีเน่+พีช ต้องเจรจาก่อนใคร เพราะเหลือเวลาน้อย + หา backup กัน single point
  3. แบ่งงาน Track A/B ให้ชัดในสัญญา — ทีมผมรับ Track B (greenfield) + บริหาร + เอกสาร · มหิดลรับ Track A + Integration + Migration · ลดการพึ่งพาข้ามทีมโดยไม่จำเป็น
  4. เคลียร์ปฏิทินกับหัวหน้าทีมมหิดล (Phase 2) — คนมหิดลยังมีงานตัวเอง · จองหนักช่วง ก.ค.–ต.ค. (สายวิเคราะห์) และ พ.ย.–มี.ค. (Dev/Integration/Migration/QA) ตาม Matrix §4
  5. Compliance: ระบุชื่อ/บทบาทจริงทั้ง 2 ทีมในเอกสารทีม (M1-10) ให้ครบ 13 ตำแหน่งตาม TOR — รวม “ยืมมหิดล” เป็นส่วนหนึ่งของทีมโครงการอย่างเป็นทางการ
⚠️ ข้อควรระวัง: Actual MD เป็นภาพ Sprint ล่าสุด ใช้เป็นสัญญาณ “ดึงง่าย/ยาก” เท่านั้น — ต้องยืนยัน availability จริงรายเดือนกับหัวหน้าทีมมหิดลก่อนล็อกแผน · ถ้าคนคอขวด (จีเน่/พีช/โอ/สาม) ยืมไม่ได้ → ต้องจ้างเสริม หรือ re-scope Track A
