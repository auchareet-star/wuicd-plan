1วัตถุประสงค์และขอบเขต

  • กำหนดการจับคู่ข้อมูลระหว่าง DIS กับระบบที่เชื่อมต่อ: HAIT Plus, SAP, LIS, PACS (และ WU-HRMS)
  • ระบุชนิดข้อมูล กฎการแปลง (transformation) และหมายเหตุของแต่ละ field
  • ใช้เป็นฐานสำหรับงาน Integration (Interface) และ Data Migration พร้อมเกณฑ์ตรวจสอบ (reconciliation)
⚠️ ข้อควรระวัง: field ที่มีกฎแปลง/รหัส master ต่างกัน ต้องตกลงกับ ทีม HIS เดิม ก่อน lock การออกแบบ (ดูหัวข้อ 3)

2ตารางจับคู่ข้อมูล (Data Mapping)

2.1 DIS ↔ HAIT Plus (ข้อมูลผู้ป่วย — Patient)

ระบบต้นทาง.fieldDIS.fieldชนิดกฎการแปลงหมายเหตุ
HAIT.PATIENT.HNdis_patient.hnVARCHAR(13)คงรูปแบบเดิม ใช้เป็น key เชื่อมPrimary key เชื่อมระบบ
HAIT.PATIENT.FNAME / LNAMEdis_patient.full_nameVARCHARต่อ FNAME + ' ' + LNAMEตรวจ encoding ภาษาไทย
HAIT.PATIENT.BIRTHdis_patient.dobDATEแปลง พ.ศ.→ค.ศ. / format YYYY-MM-DDยืนยันรูปแบบวันที่
HAIT.PATIENT.SEXdis_patient.genderCHAR(1)map 1→M, 2→F (lookup)ตกลง code list
HAIT.PATIENT.RIGHT_CODEdis_patient.coverageVARCHARmap สิทธิ์การรักษาตามตาราง lookupขอ master สิทธิ์
     

2.2 DIS ↔ SAP (การเงิน — Billing / AR)

ระบบต้นทาง.fieldDIS.fieldชนิดกฎการแปลงหมายเหตุ
DIS.invoice.invoice_noSAP.AR.DOC_NOVARCHARprefix ตามรหัสหน่วยงาน + runningส่งจาก DIS → SAP
DIS.invoice.amountSAP.AR.AMOUNTDECIMAL(12,2)หน่วยบาท ทศนิยม 2 ตำแหน่งกระทบยอด reconciliation
DIS.invoice.cost_centerSAP.AR.COST_CTRVARCHARmap คลินิก → cost centerขอตาราง cost center
DIS.invoice.gl_codeSAP.AR.GL_ACCOUNTVARCHARmap รหัสหัตถการ → ผังบัญชี (GL)ตกลงกับฝ่ายบัญชี
     

2.3 DIS ↔ LIS (ผลตรวจ — Lab Result)

ระบบต้นทาง.fieldDIS.fieldชนิดกฎการแปลงหมายเหตุ
LIS.RESULT.ORDER_IDdis_lab_order.order_idVARCHARmap ตาม HL7 ORM/ORUผูกกับ visit/HN
LIS.RESULT.TEST_CODEdis_lab_result.test_codeVARCHARmap รหัสตรวจ (lookup LOINC/local)ตกลง code list
LIS.RESULT.VALUE / UNITdis_lab_result.value / unitVARCHARคงค่า + หน่วยตามต้นทางแสดงค่าอ้างอิง (ref range)
LIS.RESULT.STATUSdis_lab_result.statusVARCHARmap P→Prelim, F→Final—
     

2.4 DIS ↔ PACS (ภาพรังสี — Imaging)

ระบบต้นทาง.fieldDIS.fieldชนิดกฎการแปลงหมายเหตุ
PACS.STUDY.STUDY_UIDdis_image.study_uidVARCHARDICOM Study Instance UID (คงค่า)ใช้ลิงก์ WADO เปิดภาพ
PACS.STUDY.PATIENT_IDdis_image.hnVARCHARmap Patient ID → HNต้องตรงกับ HAIT
PACS.STUDY.MODALITYdis_image.modalityVARCHARคงค่า (เช่น DX, PX, CBCT)—
PACS.STUDY.STUDY_DATEdis_image.study_dateDATEformat YYYY-MM-DD—
     

3รายการ field ที่ต้องตกลงกับทีม HIS เดิม

ลำดับfield / ประเด็นสิ่งที่ต้องตกลงสถานะ
1รูปแบบวันเกิด (พ.ศ./ค.ศ.)มาตรฐานการแปลงวันที่ที่ใช้ร่วมกันรอยืนยัน
2Master สิทธิ์การรักษา (RIGHT_CODE)ตาราง lookup รหัสสิทธิ์และการ mapรอยืนยัน
3Code list เพศ/สถานะค่ามาตรฐานและความหมายรอยืนยัน
4วิธีเชื่อมต่อ (API/HL7/DB View)ช่องทาง, ความถี่ sync, การ authรอยืนยัน
5รหัสหัตถการ ↔ GL/cost centerการ map ไป SAPรอยืนยัน
