1ภาพรวมการเชื่อมต่อ (Integration Overview)

DIS เชื่อมต่อกับระบบภายนอกแบบ Real-time Fully Integrated ผ่าน REST API เป็นหลัก (และรองรับ HL7/DICOM สำหรับระบบเฉพาะทาง) ตารางด้านล่างสรุปรายการ interface ทั้งหมดในโครงการ

2มาตรฐานทั่วไป (General Standards)

2.1 การยืนยันตัวตน (Authentication)

  • OAuth 2.0 (Client Credentials Grant) สำหรับ service-to-service · ส่ง Access Token แบบ Bearer ใน header
  • Token หมดอายุ 3600 วินาที · ขอ token ใหม่ผ่าน /oauth/token ของ Authorization Server
  • ทุกการเรียกต้องผ่าน HTTPS (TLS 1.2 ขึ้นไป) เท่านั้น

2.2 รูปแบบข้อมูล (Data Format)

  • Request/Response เป็น JSON (UTF-8) · Content-Type: application/json
  • วันที่-เวลาใช้ ISO 8601 (เช่น 2026-06-24T09:30:00+07:00) — ภายในเก็บเป็น ค.ศ. แปลงเป็น พ.ศ. เฉพาะตอนแสดงผล
  • รหัสอ้างอิงผู้ป่วยใช้ HN; เลขบัตรประชาชน 13 หลักเป็น string

2.3 การกำหนดเวอร์ชัน (Versioning)

ใช้ versioning แบบ URI prefix: /api/v1/... · เปลี่ยน major version เมื่อมี breaking change และคงเวอร์ชันเดิมไว้ระยะ deprecate ≥ 6 เดือน

2.4 รูปแบบข้อผิดพลาด (Error Format)

{ "error": { "code": "PATIENT_NOT_FOUND", "message": "ไม่พบข้อมูลผู้ป่วยตาม HN ที่ระบุ", "timestamp": "2026-06-24T09:30:00+07:00", "traceId": "a1b2c3d4-5678-90ef-1234-567890abcdef" } }

5การทำ Mock/Stub ระหว่างรอทีม HIS เดิม

เนื่องจากทีมผู้รับจ้างไม่มีผู้เชี่ยวชาญ HIS เดิม และต้องจองคิวทีมเดิมล่วงหน้า การพัฒนา DIS จึงเดินคู่ขนานโดยใช้ Mock API ที่ทำตาม contract เดียวกับสเปกในข้อ 3 จนกว่าจะ sync และทดสอบกับระบบจริง

ขั้นตอนรายละเอียด
1. ตกลง Contractยืนยัน schema request/response กับทีม HIS เดิม → ใช้เป็น single source of truth
2. ทำ Mock ServerMock จาก OpenAPI spec (เช่น Prism/WireMock) คืนค่าตัวอย่างตามข้อ 3
3. พัฒนาคู่ขนานDev เรียก mock ผ่าน config flag USE_MOCK=true ไม่ต้องรอระบบจริง
4. Contract Testทดสอบสัญญา (consumer-driven) กันสเปกเพี้ยนเมื่อสลับไป endpoint จริง
5. สลับเป็นจริงเมื่อจองคิวทีมเดิมได้ ทำ Integration Test แล้วปิด flag mock
# config ตัวอย่าง (สลับ mock/จริง) his.base-url = ${USE_MOCK:false} ? http://mock:4010 : https://hait.wu.ac.th his.timeout-ms = 3000 his.fallback-mock = true # ถ้า HIS ล่ม → ใช้ mock ชั่วคราว + แจ้งเตือน
แนวทาง: เก็บ OpenAPI/Swagger spec ไว้ใน repo กลาง ใช้ทั้ง generate mock, generate client และเป็นเอกสารอ้างอิงร่วมกับทีม HIS เดิม
