3การจัดการข้อมูลคุณภาพต่ำ/ตกหล่น (Data Quality)

ปัญหาแนวทางจัดการ
วันที่ผิดรูป/0000-00-00ตั้งเป็น NULL + บันทึกลง error table เพื่อให้เจ้าของงานยืนยัน
HN ซ้ำ / CID ซ้ำกฎรวมเรคคอร์ด (merge rule) + รายงานให้ยืนยันก่อนรวม
FK กำพร้า (orphan visit ไม่มี patient)กักไว้ใน quarantine, ไม่โหลดจนกว่าจะแก้ต้นทาง
รหัส (เพศ/สิทธิ/คลินิก) นอกชุดที่ mapmap → ค่า default + flag ให้ทบทวน
ข้อมูลตกหล่น (NOT NULL ว่าง)เติมค่า default ตามกติกา หรือกักไว้แก้
  

ทุกเรคคอร์ดที่ถูกแก้/กัก/ทิ้ง ต้องบันทึกเหตุผลใน migration log และสรุปยอดเสนอเจ้าของงานก่อน sign-off

4กฎการตรวจสอบหลังย้าย (Reconciliation)

การตรวจสอบวิธี/เกณฑ์ผ่าน
นับจำนวนเรคคอร์ด (row count)count source = count target + (quarantine + rejected) ทุกตาราง
ผลรวมเชิงตัวเลข (control total)SUM(fee) ต้นทาง = ปลายทาง (ภายใน rounding ที่ยอมรับ)
สุ่มเทียบราย record (sampling)สุ่ม ≥ 1% หรือ 500 เรคคอร์ด เทียบทีละ field
ความถูกต้องของรูปแบบ (validation)วันที่เป็น ค.ศ./ISO, CID checksum, FK ครบ
เกณฑ์ยอมรับรวม100% reconciliation (สอดคล้อง M1-01) — ส่วนต่างต้องอธิบายได้ทุกรายการ
  
-- ตัวอย่าง query ตรวจ row count SELECT 'patient' AS tbl, (SELECT COUNT(*) FROM src.patient) AS src_cnt, (SELECT COUNT(*) FROM dis.dis_patient) AS tgt_cnt, (SELECT COUNT(*) FROM mig.quarantine WHERE tbl='patient') AS quar, (SELECT COUNT(*) FROM mig.rejected WHERE tbl='patient') AS rej; -- เกณฑ์ผ่าน: src_cnt = tgt_cnt + quar + rej
