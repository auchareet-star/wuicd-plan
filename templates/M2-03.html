2ผังกระบวนการเดิม (As-is)

ตัวอย่าง: กระบวนการรับบริการของผู้ป่วยทันตกรรม ตั้งแต่ลงทะเบียนจนนัดครั้งถัดไป (ส่วนใหญ่ยังใช้กระดาษและบันทึกซ้ำซ้อน)

SWIMLANE (As-is) — Flow ผู้ป่วยทันตกรรม ผู้ป่วย │ (มาถึง รพ.) │ │ เวชระเบียน │ v │ [ยื่นบัตร/กรอกฟอร์มกระดาษ] --> [ค้น/เปิดแฟ้ม OPD card กระดาษ] │ │ ผู้ช่วย/ │ v พยาบาล │ [ซักประวัติ + วัดสัญญาณชีพ จดในแฟ้ม] │ │ ทันตแพทย์ │ v │ [ตรวจ/รักษา] --> <ต้องสั่งยา?> │ │ใช่ │ไม่ │ v │ ห้องยา │ [เขียนใบสั่งยากระดาษ] │ │ │ │ │ [จ่ายยา-เช็คแพ้ยาด้วยมือ]│ │ │ │ การเงิน │ v v │ [คิดเงิน-ออกใบเสร็จด้วยมือ/แยกระบบ] │ │ │ v │ [นัดครั้งถัดไป-จดในสมุดนัด] --> (กลับบ้าน)
ปัญหาที่พบ (As-is): บันทึกซ้ำซ้อนหลายจุด, เช็คแพ้ยาด้วยมือเสี่ยงพลาด, ไม่มีระบบ Recall อัตโนมัติ, ค้นประวัติ/ภาพรังสีย้อนหลังช้า, ข้อมูลไม่เชื่อมกับบัญชี

3ผังกระบวนการใหม่ (To-be — หลังใช้ DIS)

ตัวอย่าง: กระบวนการเดียวกันหลังใช้ DIS — บันทึกครั้งเดียวไหลต่อทั้งระบบ เชื่อมแพ้ยา/บัญชี/PACS และมี Recall อัตโนมัติ

SWIMLANE (To-be) — Flow ผู้ป่วยทันตกรรมบน DIS ผู้ป่วย │ (มาถึง / นัดล่วงหน้าผ่าน Patient App) │ │ เวชระเบียน │ v │ [[DIS: ค้นผู้ป่วยด้วย HN/บัตร ปชช.]] --> [[เปิด e-เวชระเบียน]] │ │ ผู้ช่วย/ │ v พยาบาล │ [[DIS: ซักประวัติ+Vital sign บันทึกครั้งเดียว]] │ │ ทันตแพทย์ │ v │ [[DIS: Dental Chart + บันทึกการรักษา]] --> <สั่งยา?> │ │ใช่ │ไม่ │ v │ ห้องยา │ [[DIS: สั่งยา -> เตือนแพ้ยา/ยาตีกันอัตโนมัติ]]│ │ │ │ │ [[DIS: ห้องยาจ่ายยา-ตัดสต็อก]] │ │ │ │ การเงิน │ v v │ [[DIS: คิดค่ารักษาอัตโนมัติ -> บัญชีลูกหนี้/SAP]] │ │ ระบบ DIS │ v │ [[DIS: สร้างนัดถัดไป + Recall อัตโนมัติ]] --> [[แจ้งเตือนผ่าน Patient App/SMS]] │ │ │ v │ (เสร็จสิ้น)
ประโยชน์ (To-be): บันทึกครั้งเดียวไหลต่อทั้งระบบ, เตือนแพ้ยา/ยาตีกันอัตโนมัติ, ค่ารักษาเข้าบัญชี/SAP อัตโนมัติ, Recall+แจ้งเตือนนัดผ่าน Mobile, เรียกดูภาพรังสี (PACS) และผล Lab ในจอเดียว

4จุดเปลี่ยน/ปรับปรุง (As-is vs To-be)

ขั้นตอนAs-is (เดิม)To-be (ใหม่)ผลที่ได้
ลงทะเบียนกรอกฟอร์มกระดาษ ค้นแฟ้มค้นด้วย HN/บัตร ปชช. เปิด e-เวชระเบียนทันทีเร็วขึ้น ลดแฟ้มหาย
ซักประวัติจดในแฟ้ม บันทึกซ้ำหลายที่บันทึกครั้งเดียวใน DIS ใช้ต่อทั้งระบบลดงานซ้ำ/คลาดเคลื่อน
สั่ง/จ่ายยาเขียนใบสั่งยา เช็คแพ้ยาด้วยมือสั่งยาใน DIS เตือนแพ้ยา/ยาตีกันอัตโนมัติปลอดภัยต่อผู้ป่วยขึ้น
เก็บเงินคิดเงินแยกระบบ/ด้วยมือคิดค่ารักษาอัตโนมัติ เชื่อมบัญชี/SAPลดข้อผิดพลาดการเงิน
นัดถัดไปจดสมุดนัด ไม่มี Recallนัด+Recall อัตโนมัติ แจ้งเตือนผ่าน Mobileดูแลต่อเนื่อง ลด no-show
    

5รายการ Workflow ทั้งหมดที่ต้องจัดทำ (Workflow Inventory)

รายการผังกระบวนการ As-is/To-be ที่ต้องจัดทำให้ครบทุกโมดูลตาม TOR — ใช้ติดตามความครบถ้วน

ลำดับโมดูล (อ้าง TOR)ชื่อ Flowสถานะ
WF-01เวชระเบียน (1.1)Flow ผู้ป่วยทันตกรรม (ลงทะเบียน→นัดถัดไป)เสร็จ
WF-02ห้องจ่ายยา/เภสัช (1.3–1.4)Flow สั่ง-จ่ายยา + เตือนแพ้ยากำลังจัดทำ
WF-03นัดหมาย/Recall (1.11)Flow นัดหมายและเรียกกลับ (Recall)กำลังจัดทำ
WF-04Consent (1.28)Flow ขอความยินยอมอิเล็กทรอนิกส์ยังไม่เริ่ม
WF-05Dental LabFlow ใบสั่งงาน Lab และติดตามสถานะยังไม่เริ่ม
WF-06LISFlow ส่งตรวจ Lab และรับผลยังไม่เริ่ม
WF-07รังสี/PACS (1.7)Flow สั่งถ่ายภาพรังสีและดูภาพยังไม่เริ่ม
WF-08บัญชีลูกหนี้/เบิกจ่าย (1.18)Flow คิดค่ารักษาและตั้งลูกหนี้ยังไม่เริ่ม
WF-09ระบบการเรียนการสอน (ส่วนที่ 2)Flow มอบหมาย/ประเมินงานนักศึกษายังไม่เริ่ม
