3วิธีการโอนย้าย (ETL: Extract / Transform / Load)

3.1 Extract

ดึงข้อมูลจาก HAIT Plus เข้า Staging โดยไม่กระทบระบบ Production (read-only / นอกเวลาทำการ)

3.2 Transform

  • Mapping ฟิลด์ต้นทาง → ปลายทาง (ตาราง mapping แยกไฟล์)
  • แปลงรูปแบบ: ปี พ.ศ. → ค.ศ., รหัส, encoding ภาษาไทย
  • ทำความสะอาดข้อมูล (Data Cleansing) — ค่าซ้ำ/ว่าง/ไม่ valid
  • ตรวจ Referential Integrity ก่อน Load

3.3 Load

นำเข้า DIS ตามลำดับ master → transaction พร้อม log การนำเข้าและ error report

HAIT Plus DB ──(Extract)──▶ Staging ──(Transform/Cleanse)──▶ Validated ──(Load)──▶ DIS DB │ │ └──────────────────────── Reconciliation (ข้อ 5) ◀────────────────────────┘

5เกณฑ์การตรวจสอบความถูกต้อง (Reconciliation)

  • นับจำนวน record ต้นทาง = ปลายทาง ทุก dataset (Record Count = 100%)
  • ตรวจยอดรวมเชิงตัวเลข (Control Total) เช่น ยอดลูกหนี้รวม
  • สุ่มตรวจรายตัว (Sampling) อย่างน้อย __% หรือ __ records/dataset
  • ตรวจ Referential Integrity (ไม่มี orphan record)
  • ตรวจฟิลด์สำคัญ: HN, ชื่อ-สกุล, วันเกิด (พ.ศ.→ค.ศ.), สิทธิการรักษา
  • เจ้าของงานรายโมดูลยืนยัน (sign-off) ผลการ reconcile

8ความเสี่ยงและแผนถอยกลับ (Rollback)

ความเสี่ยงโอกาสผลกระทบการรับมือ
